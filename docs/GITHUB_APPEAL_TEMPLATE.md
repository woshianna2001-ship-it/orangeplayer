# GitHub 申诉模板

如果你的仓库被误判为违规，可以通过以下方式申诉：

## 📧 联系 GitHub 支持

**方式 1：通过 GitHub Support**
1. 访问：https://support.github.com/contact
2. 选择 "Account and Profile" → "Content Removal"
3. 填写表单

**方式 2：发送邮件**
- 收件人：support@github.com
- 主题：Appeal for Content Violation Flag on Repository

## 📝 申诉邮件模板（英文）

```
Subject: Appeal for Content Violation Flag on Repository orangeplayer

Dear GitHub Support Team,

I am writing to appeal the content violation flag on my repository:
https://github.com/706412584/orangeplayer

Repository Description:
OrangePlayer is an open-source Android video player library that provides features such as:
- Multiple playback engines (System/ExoPlayer/IJK/AliPlayer)
- Subtitle system (SRT/ASS/VTT formats)
- Danmaku (bullet comments) support
- OCR subtitle recognition
- Speech recognition for real-time subtitles

This is a legitimate open-source project for Android developers, licensed under Apache 2.0. The repository contains no illegal, harmful, or inappropriate content.

I believe this flag was applied in error by an automated system. Could you please review this case and remove the violation flag?

Repository Information:
- Repository: https://github.com/706412584/orangeplayer
- License: Apache License 2.0
- Purpose: Android video player library for developers
- Language: Java/Kotlin

Thank you for your time and consideration.

Best regards,
[Your Name]
GitHub Username: 706412584
```

## ⏱️ 预计处理时间

- 通常 1-3 个工作日内会收到回复
- 如果是误判，通常会在 24 小时内解除

## 📋 申诉时需要准备的信息

1. 仓库 URL
2. 项目说明
3. 开源许可证
4. 项目用途
5. 为什么认为是误判

## 🔄 申诉后的跟进

如果 3 个工作日内没有回复：
1. 再次发送邮件
2. 在 Twitter 上 @GitHubSupport
3. 在 GitHub Community 论坛发帖

## 💡 预防措施

为避免再次被误判：
1. 在仓库描述中明确说明这是开发者工具
2. 添加详细的 README 说明项目用途
3. 添加开源许可证
4. 避免使用可能被误解的词汇
